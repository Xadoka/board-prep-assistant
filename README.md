# Board Prep Assistant

Ассистент подготовки к совету директоров. Собирает данные из разнородных источников
(финансовые/операционные отчёты, транскрипты созвонов, переписка) и превращает их в единый
**board packet**: executive summary, таблицу ключевых метрик с динамикой, список расхождений
и вопросы для обсуждения.

**Ключевой принцип:** каждая цифра в packet прослеживается до источника (документ + дословная
цитата). Извлечение фактов отделено от синтеза — модель сначала вытаскивает проверяемые факты
с цитатами, и только потом синтезирует материалы из проверенных фактов.

Полная спецификация — в [`PRD.md`](PRD.md).

## 🔗 Демо

Живой интерактивный прототип (GitHub Pages): **https://xadoka.github.io/board-prep-assistant/**

Прототип реализует все четыре экрана из §7 PRD на реалистичных демо-данных:

| Экран | Что показывает |
|-------|----------------|
| **Дашборд** | список подготовок и их статусов, сводные счётчики |
| **Источники** | drag-and-drop загрузка, список документов со статусами (обработан / ошибка), превью |
| **Ревью фактов** | таблица фактов с цитатами и уверенностью; расхождения подсвечены, низкая уверенность приглушена; факты можно править и удалять до синтеза |
| **Board packet** | executive summary, метрики с дельтами, блок расхождений, вопросы — каждая цифра кликабельна и ведёт к цитате |

Светлая/тёмная тема, плоский дизайн, sentence case — по визуальному эталону из PRD.

## Статус реализации

Этот репозиторий содержит **фронтенд-прототип** (статический, без сборки) —
он деплоится на GitHub Pages и демонстрирует продукт и UX сквозного пайплайна.

Полноценный бэкенд по §5/§11 PRD (RAG-конвейер) — следующий шаг и **не** входит в статический
деплой, поскольку требует серверного рантайма, БД и ключа Anthropic API:

- **Фронт + бэк:** Next.js (App Router, TypeScript), API routes
- **БД:** Postgres + pgvector
- **LLM:** Anthropic Claude API (`claude-sonnet-4-6`) — извлечение фактов и синтез
- **Хранилище:** локальная папка `./storage` (интерфейс абстрагирован под S3)
- **Очередь:** in-process воркер на MVP

Порядок разработки бэкенда — вертикальными срезами (§8 PRD): сквозной MVP на одном PDF →
форматы (XLSX/DOCX/транскрипты) → почта → семантический поиск → реконсиляция и вопросы → экспорт.

## Локальный запуск прототипа

Это один статический файл — сервер не нужен:

```bash
# просто откройте в браузере
open index.html          # macOS
start index.html         # Windows

# либо через простой сервер
npx serve .
```
